## 技能目标
根据用户输入的参数，从项目根目录中的 `JOB_LIST.csv`（文档 A）和 `入出力一覧.csv`（文档 B）中检索匹配数据，执行清洗与格式化转换，最终输出标准化的作业配置格式。

---

## 输入参数与预处理
AI 必须首先解析用户提供的以下参数，并严格按照下表规则处理缺失值：
| 参数名 | 必填 | 默认值 / 缺失处理 | 说明 |
| :--- | :---: | :--- | :--- |
| **Mode** | 是 | - | 执行模式（固定为 `PTN2`） |
| **JOB_ID** | 是 | - | **唯一检索 Key**（对应两份 CSV の `[ジョブ名]` 列） |
| **CFDB_ID** | 否 | `[CFDB待采番]` | 第一部分（Part 1）の作业编号（输出占位符） |
| **TFSM_ID** | 否 | `[TFSM待采番]` | 第二部分（Part 2）の作业编号（输出占位符） |
| **TARGET_DB** | 否 | `None` | 多表输入时の目标数据库名 |

### ⚠️ 关键说明
1. **JOB_ID 是唯一检索标识**：CSV 中查找的实际数据必须满足条件**JOB_ID=ジョブ名列的值**
2. **CFDB_ID 和 TFSM_ID 是输出占位符**：用于替换输出格式中的作业编号，**不参与 CSV 检索**
3. **参数顺序不可颠倒**：必须严格按照 `PTN2 <JOB_ID> [CFDB_ID] [TFSM_ID] [TARGET_DB]` 顺序解析

### ⚠️ 强制执行规则 - 检索阶段（必须严格遵守）
1. **仅使用第二个参数（JOB_ID）进行检索**：必须且只能使用 `JOB_ID` 参数在 CSV 中查找数据
2. **禁止使用其他参数进行检索**：`CFDB_ID`、`TFSM_ID`、`TARGET_DB` 仅用于输出占位符，**绝对禁止**用于 CSV 检索
3. **禁止使用任何其他作业ID进行检索**：用户输入的其他参数（如 SSF_CFUD0099、SSF_TFSM0099、REQUEST_DISPATCH_PAPERS）**绝对禁止**用于检索
4. **单次检索原则**：每次执行必须且只能检索一次，使用唯一的 JOB_ID 参数

## PTN2 核心处理逻辑

### 1. 数据检索规则（必须严格执行）
*   **文档 A (`JOB_LIST.csv`) 检索**：
    *   **检索条件**：`[ジョブ名]` 等于 `JOB_ID`（仅使用第二个参数）
    *   **提取字段**：`連絡方法（異常）`、`ジョブの処理内容説明`、`開始時刻`、`稼働日スケジュール`、`先行ジョブ`
    *   **⚠️ 强制约束**：必须且只能检索一次，使用 JOB_ID 参数
*   **文档 B (`入出力一覧.csv`) 检索**：
    *   **检索条件**：`[ジョブ名]` 等于 `JOB_ID` **且** `[Inout]` 字段等于 `"IN"`
    *   **提取字段**：`説明`、`データ名`、`サーバ名・インスタンス名`、`ディレクトリ名・スキーマ名`
    *   **⚠️ 强制约束**：必须且只能检索一次，使用 JOB_ID 参数
    *   *注意：若匹配到多条 IN 记录，需完整提取并在输出の `IN` 数组中循环列出。*

### 2. 数据清洗与加工规则
